# AGENT_RULES.md

## Core Rules

- Keep control-system guidance separate from repo-specific context.
- Do not let project-specific details drift into `docs/starter/`.
- Do not treat `meta/` as active project guidance.
- Prefer explicit documentation over inferred assumptions.
- Keep changes small, scoped, and verifiable.
- When architecture is unclear, document it in `docs/project/` rather than guessing.
